Step-by-Step Setup Guide
Step 1 of 5
Register an App in Azure Active Directory
This creates the "identity" that lets the tool talk to your SharePoint. You only do this once.
  1. Go to portal.azure.com and sign in with your Microsoft 365 admin account
  2. Search for "App registrations" and click + New registration
  3. Name it: IL SNF Rate Optimizer
  4. Supported account types: "Accounts in this organizational directory only"
  5. Redirect URI: Select Single-page application (SPA) and enter:
    https://aperion.sharepoint.com/sites/RegionalReimbursementTeam/SiteAssets/snf-optimizer.html
    (Also add http://localhost for local testing)
  6. Click Register
  7. Copy the Application (client) ID and Directory (tenant) ID — paste them in the Connection Settings above
Step 2 of 5
Grant API Permissions
  1. In your new app registration, click API permissions → + Add a permission
  2. Choose Microsoft Graph → Delegated permissions
  3. Search and add these permissions:
    Sites.ReadWrite.All User.Read
  4. Click Grant admin consent for Aperion (requires admin privileges)
  5. Both permissions should show a green ✓ "Granted"
Step 3 of 5
Create the SharePoint List
Create a list called SNF_Weekly_Tracker at your SharePoint site:
  1. Go to aperion.sharepoint.com/sites/RegionalReimbursementTeam
  2. Click + New → List → Blank list
  3. Name it exactly: SNF_Weekly_Tracker
  4. Add these columns (click + Add column for each):
    WeekEnd → Date (Date only) FacilityName → Single line of text CMIGoal → Number (2 decimal places) MedAGoal → Number (0 decimal places) StriveCMIGoal → Number (2 decimal places) CMI → Number (2 decimal places) MedARate → Number (0 decimal places) hPPD → Number (2 decimal places) StrivePct → Number (2 decimal places) StriveDollar → Number (2 decimal places) Bottom4 → Number (0 decimal places) RatePeriod → Single line of text Notes → Multiple lines of text
  5. The default Title column can be kept — it will store a unique key (FacilityName + WeekEnd)
Column names must match exactly (case-sensitive). The tool uses these names to read and write data.
Step 4 of 5
Upload This File to SharePoint
  1. In your SharePoint site, go to Site Contents → Site Assets library
  2. Upload il_snf_optimizer_sp.html to Site Assets
  3. The file will be accessible at:
    https://aperion.sharepoint.com/sites/RegionalReimbursementTeam/SiteAssets/il_snf_optimizer_sp.html
  4. To embed it on a SharePoint page:
    • Edit any SharePoint page → + → search "Embed"
    • Paste the URL above into the embed URL field
    • The tool renders inline with full functionality
Step 5 of 5
Connect and Test
  1. Fill in the Client ID and Tenant ID in Connection Settings above
  2. Click Connect & Sign In — a Microsoft login window will appear
  3. Sign in with your Aperion Microsoft 365 credentials
  4. Click Test Connection to verify the list is reachable
  5. Click ↑ Sync to SP in the status bar to push your local data up
  6. Share the SharePoint page URL with your consultants and RVPs — they sign in with their own credentials and can enter data for their facilities
Once connected: Every time someone edits a cell in the Weekly Tracker, the change auto-saves to the SharePoint list if Auto-sync is checked above. Your whole team always sees current data.
Optional: Power BI Dashboard
Build a Live Dashboard from Your SharePoint List
Once data is in the SNF_Weekly_Tracker list, you can connect Power BI directly:
  1. Open Power BI Desktop → Get Data → SharePoint Online List
  2. Enter site URL: https://aperion.sharepoint.com/sites/RegionalReimbursementTeam
  3. Select SNF_Weekly_Tracker list
  4. Build visuals for CMI trend by facility, STRIVE % over time, Med A rate heat map
  5. Publish to Power BI Service and embed on the same SharePoint page next to the tracker
